- Model :
   -> Distance
   -> Parametre
- Repository :
   -> DistanceRepository
      -> int getDistance(int idHotelA, int idHotelB)
         Chercher A → B sinon chercher B → A et retourner la distance
   -> ParametreRepository
      -> String getValeurByCode(String code)
- Service :
   -> ParametreService
      -> int getVm()
         Appeler ParametreRepository.getValeurByCode("Vm") et convertir en int
   -> DureeService
      -> DureeResult calculerDuree(LocalDateTime dateArrivee, int idHotel)
      Pour 
         distance = DistanceRepository.getDistance(Aeroport, idHotel)
         Vm = ParametreRepository.getVm()
         dureeMinutes = (distance / Vm) * 60 * 2
         dateDepart = dateArrivee
         dateRetour = dateDepart + dureeMinutes
- Util :
   -> DureeResult
      -> LocalDateTime dateDepart
      -> LocalDateTime dateRetour
      -> int dureeMinutes

Branche 1 : sprint3/2/assignation_service
- Model :   
   -> Assignation
   -> Vehicule 
   -> Reservation
   -> classe de DEV 1 (Distance, Parametre)
- Repository :
   -> AssignationRepository
      -> void insertAssignation(Assignation assignation)
      -> boolean existsAssignationForDate(LocalDate date)
   -> VehiculeRepository
      -> List<Vehicule> getVehiculesDisponible(int nbPassager, LocalDateTime dateDepart, LocalDateTime dateRetour)
         Condition :
            nbPlace >= nbPassager
            ET
            NOT (dateRetour <= ancien_debut OR dateDepart >= ancien_fin)
   -> ReservationRepository
      -> List<Reservation> getReservationsByDate(LocalDate date)
         => Conversion interne :
            LocalDateTime debut = date.atStartOfDay();
            LocalDateTime fin = date.atTime(23,59,59);
         => Requête SQL :
            WHERE DateHeureArrive BETWEEN :debut AND :fin ORDER BY DateHeureArrive ASC
- Service :
   -> AssignationService
      -> Boolean checkAssignationExistsForDate(LocalDate date)
         => Appeler AssignationRepository.existsAssignationForDate(date)
      -> AssignationResult assignerPourDate(LocalDate date)
      Pour 
         Si checkAssignationExistsForDate(date) == true
         throw new IllegalStateException("Une assignation a deja ete effectue pour cette date")
         BEGIN TRANSACTION
            reservations = ReservationRepository.getReservationsByDate(date)
            Pour chaque reservation R dans reservations
               dureeResult = DureeService.calculerDuree(R.DateHeureArrive, R.Id_Hotel)
               dateDepart = dureeResult.dateDepart
               dateRetour = dureeResult.dateRetour

               vehiculesDisponibles = VehiculeRepository.getVehiculesDisponible(R.nbPassager, dateDepart, dateRetour)

               Si vehiculesDisponibles est vide
                  -> Ajouter R dans listeNonAssignee
                  -> Continuer boucle

               Appliquer priorisation sur vehiculesDisponibles
                  Étape 1 : Capacité minimale suffisante
                     -> garder les véhicules avec nbPlace le plus proche possible (MIN(nbPlace) >= R.nbPassager)
                  Étape 2 : Diesel prioritaire
                     -> Si parmi ces véhicules il existe au moins un TypeVehicule = 'D'
                        -> garder uniquement les Diesel
                     -> Sinon
                        -> garder tous les véhicules restants
                  Étape 3 : Random
                     -> sélection aléatoire
                        (PostgreSQL : ORDER BY RANDOM() LIMIT 1)
                     
               Vehicule choisi = le véhicule qui reste après application de la priorisation

               Créer Assignation
               insertAssignation(assignation)
         COMMIT
         ROLLBACK si erreur
- Util :
   -> AssignationResult
      -> List<Assignation> assignations
      -> List<Reservation> reservationsNonAssignees
