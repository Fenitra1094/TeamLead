Branche 1 :  sprint6/1/vehicule_repository
- Modifier `VehiculeRepository.getVehiculesDisponible()` :
Objectif :
    Retourner les vehicules candidats pour une reservation, en tenant compte :
    1) du nombre de trajets deja faits dans la date demandee
    2) des vehicules qui reviennent pendant l'intervalle du groupe

  Entrees :
    - nbPassagers
    - date (date d'assignation)
    - debut_groupe (heure courante de depart du groupe)
    - fin_groupe
    - connection

  Algorithme (repository) :

    Etape 1 - Sous-requete "charge du jour" (date parametree)
      SELECT t.Id_Vehicule,
             COUNT(t.Id_Trajet) AS trajet_count
      FROM dev.Trajet t
      WHERE DATE(t.date_assignation) = ?
      GROUP BY t.Id_Vehicule

    Etape 2 - Requete principale
      SELECT v.*,
             COALESCE(tc.trajet_count, 0) AS trajet_count
      FROM dev.Vehicule v
      LEFT JOIN (
        -- Etape 1
      ) tc ON tc.Id_Vehicule = v.Id_Vehicule
      WHERE v.nbPlace >= ?
        AND (
          -- Cas A : vehicule deja libre au debut_groupe
          dernier_retour <= debut_groupe
          OR
          -- Cas B : vehicule qui revient pendant le groupe
          (dernier_retour > debut_groupe AND dernier_retour <= fin_groupe)
        )
      ORDER BY
        trajet_count ASC,
        CASE WHEN v.typeVehicule = 'D' THEN 0 ELSE 1 END ASC,
        RANDOM()

    Etape 3 - Retour methode
      Mapper les lignes SQL vers List<Vehicule> (avec trajet_count),
      puis laisser le service choisir le premier candidat.

  Signature conseillee :
    getVehiculesDisponible(int nbPassagers, LocalDate date, LocalDateTime debut_groupe, LocalDateTime fin_groupe, Connection conn)
    => retourne les vehicules capacitaires qui sont libres ou qui reviennent avant fin_groupe

Branche 2 : sprint6/1/assignation_service
- Adapter `AssignationService.assignerPourDate()` pour la logique retour :

  Algorithme par groupe :
  
    reservationsRestantes = groupe.reservations + reservations_reportees_groupes_precedents
    dateDepart = groupe.getHeureDepartGroupe()
    fin_groupe = premiere_reservation_du_groupe + tempsAttenteMinutes
    
    TANT QUE reservationsRestantes non vide :
      R = reservationsRestantes (plus grand nbPassager)
      candidats = getVehiculesDisponible(R.nbPassager, date, fin_groupe, conn)
      
      SI candidats vide :
        marquer R comme "a reporter" => passer a la suivante
        SI toutes les R ont ete tentees sans succes => sortir boucle
      
      SINON :
        vehicule = candidats.get(0)  // best candidate (trajet_count + diesel + rand)
        dateDispoVehicule = MAX(dateDepart, vehicule.dateRetourDernierTrajet)
        dateDepart = MAX(dateDepart, dateDispoVehicule)  // mise a jour depart groupe
        
        Remplir vehicule : R + autres reservations compatibles (passager <= place_restante)
        Creer Trajet(dateHeureDepart = dateDepart)
        Creer Assignations(dateHeureDepart = dateDepart)
        Retirer reservations assignees de reservationsRestantes

    IMPORTANT : dateHeureDepart est mise a jour seulement si le vehicule est effectivement assigne a une reservation, pas juste candidat. -> izany hoe raisina ilay lera nahatongavan'ilay vehicule rehefa toa ka izy no lera > indrindra de raha assigne amina reservation ao anatin'ilay groupe izy 
    
    reservations_reportees = reservationsRestantes (portees au groupe suivant)
    SI dernier groupe : elles deviennent reservationsNonAssignees
