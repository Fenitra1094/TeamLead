--------------------------------------------------------------------------------

      -> int idTrajetEtape
      -> int idTrajet
      -> int idHotel
      -> int ordre
      -> int distanceDepuisPrecedent
      -> Hotel hotel                 // Pour l'affichage

- Repository :
   -> TrajetRepository
      -> int insertTrajet(Trajet trajet, Connection conn)
         Retourner l'id généré (RETURNING id_trajet)
      -> void insertTrajetEtape(TrajetEtape etape, Connection conn)
      -> List<Trajet> findTrajetsByDate(LocalDate date)
         Avec jointures pour charger véhicule et étapes

- Service :
   -> RouteService
      -> List<TrajetEtape> calculerRoute(List<Integer> idHotels)
         Algorithme nearest-first avec tie-breaking lexicographique :
            1. Point de départ = Aéroport (ID_AEROPORT = 1)
            2. Parmi les hôtels restants, trouver le plus proche du point courant
               -> Appeler DistanceRepository.getDistance(pointCourant, hotel)
            3. Si plusieurs hôtels ont la même distance minimale (RG11) :
               -> Comparer les noms (libellés) des hôtels par ordre lexicographique
               -> Choisir celui dont le nom vient en premier alphabétiquement
               -> Ex: distance Aéroport→Panache = 15 km et Aéroport→Pachniko = 15 km
                  -> Comparer "Pachniko" vs "Panache"
                  -> Les 2 premiers caractères sont identiques "Pa"
                  -> 3ème caractère : 'c' < 'n'
                  -> Donc Pachniko passe en premier
            4. Ajouter cet hôtel comme prochaine étape (avec ordre et distance)
            5. Ce hôtel devient le nouveau point courant
            6. Répéter jusqu'à avoir visité tous les hôtels
            7. Retour : ajouter distance du dernier hôtel vers l'aéroport
         Retourner la liste ordonnée des étapes

         // Pour l'implémentation : utiliser String.compareTo() de Java
         // qui fait exactement la comparaison lexicographique souhaitée
         // hotelA.getNom().compareTo(hotelB.getNom()) < 0 → A passe avant B

      -> int calculerDistanceTotale(List<TrajetEtape> etapes)
         Somme de toutes les distanceDepuisPrecedent (y compris retour aéroport)

   -> Modifier DureeService :
      -> Ajouter méthode :
         DureeResult calculerDureeMultiStop(LocalDateTime dateDepart, List<TrajetEtape> etapes)
         Pour
            distanceTotale = somme des distances de toutes les étapes
            Vm = ParametreService.getVm()
            dureeMinutes = (distanceTotale / Vm) * 60
            dateRetour = dateDepart + dureeMinutes
         // Note : pas de * 2 car la distance totale inclut déjà l'aller ET le retour

- Util :
   -> Conserver DureeResult existant (pas de modification)

Branche 1 : sprint4/2/enhanced_assignation_service
--------------------------------------------------------------------------------
- Model :

   -> Modifier Assignation :

- Util :
   -> GroupeVol (nouvelle classe utilitaire)
      -> LocalDateTime heureArrivee    // Heure d'arrivée du vol (commune à tous)
      -> List<Reservation> reservations // Réservations de ce vol, triées par nbPassager DESC

   -> Modifier AssignationResult :

      -> Ajouter getter/setter
      // Pour afficher les détails des trajets (véhicule, étapes, horaires)

- Service :
   -> GroupingService (nouveau)
      -> List<GroupeVol> grouperParVol(List<Reservation> reservations)
         Pour
            Trier les réservations par DateHeureArrive ASC

            groupes = nouvelle liste vide

            Pour chaque reservation R :
               Chercher dans groupes un GroupeVol avec heureArrivee == R.DateHeureArrive
               // Comparaison exacte : même date, même heure, même minute
               Si trouvé :
                  -> Ajouter R dans ce groupe
               Sinon :
                  -> Créer nouveau GroupeVol
                     heureArrivee = R.DateHeureArrive
                  -> Ajouter R
                  -> Ajouter à groupes

            Pour chaque groupe :
               Trier reservations par nbPassager DESC (RG7)

            Retourner groupes

   -> Modifier AssignationService :
      -> Remplacer la méthode assignerPourDate(LocalDate date) :
         Pour
            Si checkAssignationExistsForDate(date) == true
               throw new IllegalStateException("...")

            BEGIN TRANSACTION

            reservations = ReservationRepository.getReservationsByDate(date)
            groupes = GroupingService.grouperParVol(reservations)

            Pour chaque GroupeVol groupe :
               dateDepart = groupe.heureArrivee  // Le véhicule part à l'heure d'arrivée du vol

               // Les réservations sont déjà triées par nbPassager DESC (RG7)
               Pour chaque Reservation R dans groupe.reservations (non encore assignée) :

                  vehiculesDisponibles = VehiculeRepository.getVehiculesDisponible(
                     R.nbPassager, dateDepart, dateRetourEstime)
                  // Note : dateRetourEstime = estimation large pour la requête,
                  //        sera recalculé précisément après le routage

                  Appliquer priorisation RG1-RG4 sur vehiculesDisponibles
                  Si aucun véhicule → ajouter R à listeNonAssignee, continuer

                  vehiculeChoisi = véhicule retenu

                  // RG8 : Remplissage optimal
                  placesRestantes = vehiculeChoisi.nbPlace - R.nbPassager
                  reservationsDansVehicule = [R]
                  hotelsDansVehicule = [R.hotel.idHotel]

                  Pour chaque Reservation R2 restante dans groupe (non assignée, R2 != R) :
                     Si R2.nbPassager <= placesRestantes
                        -> Ajouter R2 dans reservationsDansVehicule
                        -> placesRestantes -= R2.nbPassager
                        -> Ajouter R2.hotel.idHotel dans hotelsDansVehicule (si pas déjà)
                        -> Marquer R2 comme assignée dans ce tour

                  // RG9 + RG11 : Calculer la route optimale (nearest-first + lexicographique si égalité)
                  etapes = RouteService.calculerRoute(hotelsDansVehicule)
                  dureeResult = DureeService.calculerDureeMultiStop(dateDepart, etapes)

                  // Créer le Trajet
                  trajet = nouveau Trajet(vehiculeChoisi, dateDepart, dureeResult.dateRetour, date)
                  idTrajet = TrajetRepository.insertTrajet(trajet, connection)
                  Pour chaque etape : TrajetRepository.insertTrajetEtape(etape, connection)

                  // Créer les Assignations pour chaque réservation du véhicule
                  Pour chaque Reservation Rx dans reservationsDansVehicule :
                     assignation = nouvelle Assignation(Rx, vehiculeChoisi, dateDepart,
                                                        dureeResult.dateRetour, idTrajet)
                     AssignationRepository.insertAssignation(assignation, connection)

            COMMIT
            ROLLBACK si erreur

            Retourner AssignationResult(assignations, nonAssignees, trajets)

   -> Modifier VehiculeRepository :
      -> Modifier getVehiculesDisponible :
         La vérification de disponibilité doit maintenant checker la table Trajet
         au lieu de (ou en plus de) la table Assignation
         Car un véhicule est indisponible pendant toute la durée de son trajet

         SELECT * FROM Vehicule v
         WHERE v.nb_place >= :nbPassager
         AND v.id_vehicule NOT IN (
            SELECT t.id_vehicule FROM Trajet t
            WHERE NOT (t.date_heure_retour <= :dateDepart OR t.date_heure_depart >= :dateRetour)
         )
         ORDER BY v.nb_place ASC,
                  CASE WHEN v.type_vehicule = 'D' THEN 0 ELSE 1 END,
                  RANDOM()
